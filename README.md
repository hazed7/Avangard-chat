# Avangard Chat

Avangard Chat представляет собой кроссплатформенный веб-мессенджер, разработанный для обеспечения отказоустойчивой коммуникации в условиях ограничений внешних сервисов. Проект ориентирован на создание минималистичного и производительного интерфейса, поддерживающего групповое взаимодействие, обмен медиафайлами и специализированные функции на базе искусственного интеллекта.

## Проблематика и целевое назначение

Проект разработан как ответ на текущую рыночную ситуацию в РФ, характеризующуюся блокировками иностранных мессенджеров и отсутствием популярных отечественных альтернатив с современным пользовательским интерфейсом. Целевой аудиторией продукта являются пользователи, нуждающиеся в надежном инструменте для обмена сообщениями, не имеющем избыточной функциональности социальных сетей.

В процессе анализа конкурентов были выявлены и учтены критические недостатки существующих систем:
* Telegram: Перегруженность функциоанлом и блокировка на территории РФ  
* WhatsApp: Устаревший дизайн и жесткие лимиты на объем передаваемых данных, и Блокировка на территории РФ.
* ВКонтакте: Избыточность интерфейса и общая перегруженность платформы. 
* MAX: Обязательная интеграция с государственными порталами и наличие системных ошибок. 

## Функциональные характеристики

### Основной функционал
* Система многофакторной авторизации пользователей.
* Организация персональных и групповых чатов в режиме реального времени.
* Система обмена файлами различных форматов.
* Интегрированный поиск по архиву сообщений и метаданным файлов.
* Инструменты модерации и блокировки нежелательных пользователей.

### Специальные возможности (Accessibility)
* Система синтеза речи для озвучивания входящих текстовых уведомлений.
* Голосовой ввод текста для упрощения процесса набора сообщений.
* Динамическая настройка параметров шрифтов и контрастности интерфейса.

### Модули искусственного интеллекта
* Алгоритмическое обнаружение и превентивная блокировка спам-рассылок.
* Автоматизированный перевод текстовых сообщений на лету.
* Генерация краткой выжимки (summary) на основе истории непрочитанных сообщений в группах.
* Интеллектуальная корректировка стилистики и грамматики набираемого текста.

## Технологический стек и обоснование

Архитектура системы построена на современных высокопроизводительных решениях, обеспечивающих масштабируемость и низкие задержки при передаче данных.

### Серверная часть (Backend)
* FastAPI: Использование асинхронного фреймворка на базе Python обеспечивает высокую пропускную способность при обработке I/O-интенсивных задач.
* WebSockets: Протокол задействован для реализации полнодуплексной связи, что является критическим требованием для мгновенной доставки сообщений.
* MongoDB: Документоориентированная СУБД выбрана для гибкого хранения истории сообщений и метаданных без жестко заданной схемы.
* Dragonfly: Применяется в качестве высокопроизводительного in-memory хранилища для кэширования данных и управления состояниями пользовательских сессий.
* Minio (S3): Обеспечивает надежное хранение пользовательского контента (изображения, аудиосообщения) через S3-совместимый API.
* Typesense: Интегрирован для обеспечения мгновенного полнотекстового поиска с поддержкой опечаток.

### Клиентская часть (Frontend)
* React и TypeScript: Обеспечивают строгую типизацию данных и компонентный подход к построению интерфейса.
* Макет: Дизайн выполнен в темной цветовой схеме с использованием принципов Glassmorphism для минимизации визуальной нагрузки на пользователя.

## Инструкция по развертыванию

### Системные требования
* Docker Engine 24.0.0+
* Docker Compose 2.20.0+

### Процедура установки
1. Выполнить клонирование репозитория:
   `git clone https://github.com/DiM3NT0R/Avangard-chat.git`
2. Перейти в директорию проекта:
   `cd Avangard-chat`
3. Создать файл конфигурации окружения на основе примера:
   `cp .env.example .env`
4. Запустить сборку и развертывание контейнеров:
   `docker-compose up -d --build`

## Конфигурация системы

Настройка параметров осуществляется через переменные окружения в файле `.env`. Основные параметры:
* MONGO_URI — строка подключения к базе данных MongoDB.
* DRAGONFLY_URL — адрес сервера Dragonfly.
* S3_ENDPOINT — точка доступа к хранилищу Minio.
* TYPESENSE_API_KEY — ключ доступа к поисковому движку.

## Документация API

После успешного запуска серверной части полная спецификация методов API в формате OpenAPI доступна по адресу:
`http://localhost:8000/docs` (интерфейс Swagger).
